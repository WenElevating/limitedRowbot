## 1. 触发条件
- 每当对系统功能进行 **创建（Create）、读取（Read）、更新（Update）、删除（Delete）** 操作时，必须更新 `CHANGELOG.md`。

## 2. 操作要求
- **Create / Update / Delete** 操作：
  - 操作完成后，必须在 `CHANGELOG.md` 添加一条记录。
  - 记录应包含：
    - 功能或模块名称
    - 操作类型（新增 / 修改 / 删除）
    - 简短描述本次变更
    - 操作时间（可选）

- **Read** 操作：
  - 如果涉及重要查询逻辑调整或新增接口，也应记录在 CHANGELOG 中。

## 3. 验证要求
- **提交前检查**：CI/CD 或代码审核流程必须验证本次修改是否包含对应的 `CHANGELOG.md` 更新。
- **缺失记录阻止合并**：若 CRUD 操作未在 CHANGELOG 中体现，则阻止合并或部署。

## 4. 格式标准
- 遵循现有 CHANGELOG.md 约定。
- 示例：
  - 新增：2026-02-22 | 用户管理模块 | Create | 新增用户角色权限接口
  - 修改：2026-02-22 | 用户管理模块 | Update | 修改用户角色权限验证逻辑
  - 删除：2026-02-22 | 用户管理模块 | Delete | 删除过期用户角色接口
